# Speculative Decoding (投机采样)：跳出内存墙的魔手

> **面试一句话绝杀**：当我们被极低的算力（TFlops）利用率折磨到想要跳楼时，大家都意识到了**大模型不是在算，而是在读**！也就是大模型的生成速度是被“主内存往芯片传输权重的宽带（Memory Bandwidth Bound）”死死卡住的！为了打破这个枷锁，Google 提出的投机采样（Speculative Decoding）让一个小模型在前方跑腿生成草稿，大模型在后方发起并行检验。这样让**验证几个词的代价无限逼近于生成一个词的代价**！这就是白嫖！

### 一. 内存墙惨案：为什么算得快却出不来字？

大模型做一次前向也就是生成一个字（Batch=1 时），对于 A100 来讲：
*   算数学题（乘加运算）只需要 0.0001 秒
*   把大模型身上那 140 GB 恶心的参数块从远方搬运到算力核里需要：0.05 秒。
你可以极其清晰地看出来，**绝大部分的毫秒全部拿去等公交车过来了！大家都是苦苦坐等！这也就是内存阻塞上限！**。
这时候如果有神仙说：我们有没有可能，等这趟极其慢的车过来时，不要只拉 1 个人（1个字），而是直接拉一堆人（验证 5 个字）？因为对于这种变态级的并行怪兽矩阵，横向加五个人的向量一起进去算，因为是极其恐怖的矩阵相叠并发，它花的时间也是 0.05 秒！完全零额外开销！

### 二. 双剑合璧：起搏跟进校验网络

> 代码参考文件：`speculative_decoding.py`

这就催生出了今天极度极度常用的 **Draft And Verify 机制**：
我们搞一个小杂兵（比如只有十分之一大的模型）。它的权重极少，提取时间直接闪现只需要 0.005  秒，它直接像无头苍蝇一样疯狂连写 5 个词。
接着把这个成果连带它身上本来自带的算出的概率一起提交大局座。大模型在做 0.05 秒极漫长的全身权重提取后，做一发恐怖的宏观横向判定。
如果小蚂蚁非常走运跟老大哥脑波在某一区段合拢（这叫做 Probability Alignment）。直接通吃拿下方块中的前 $N$ 个字，大步甩开由于 Auto-Regressive 必须逐词死读权重的地狱宿命！
