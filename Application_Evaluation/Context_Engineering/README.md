### 一. DSPy 极道：将 Prompt 作为“张量”跑优化策略

以前大家学怎么让大模型变聪明，都是学去给大模型磕头：写长篇大论的身份声明、编排极其复杂的少样本示例 (Few-Shot examples)。
*   **脆弱期（过时玩法）**：团队用 GPT-4 把提示词写得天花乱坠完美上线。老板由于成本考虑换成了 Qwen-Max，整个提取管线突然尿崩了。你只能继续无头苍蝇一样人工手拼改 prompt。

*   **DSPy 时代代码制霸** (`dspy_programmatic.py`)：
斯坦福这群疯子提出：**Prompt 根本就不应该由人来硬写。**
1. **Signature (签名式约定)**：你只要在代码里写个骨架：`class QA(dspy.Signature): context = dspy.InputField() -> answer = dspy.OutputField()`
2. **Teleprompter (神级调配编译器）**：你提供 100 道带答案的小练兵测试集题，随便选用一个超级外援老师（比如 GPT-4o 当评委）。
编译器就会在后台发起成百上千次的闭环推理，每一次它都会提取一些它觉得最有代表性的那几个例子（少数优质 few-shots），并自己摸索修改它最想看到的指示词，挂接到你的管道上！
3. **彻底隔离系统解耦**：从此不论怎么换基座大模型接口，你不需要动你脑残的那些冗长英语。只需要调一行语句重新 `compile(dataset)`，让死板愚蠢的机器去帮你寻找对应那个新模型最喜欢的尿性输入口味。这就是最前沿的程序式应用工程！

### 二. RAG 的致命弱点与图算法降维打击

普通向量 RAG (Retrieval-Augmented Generation) 是极其愚蠢的：拿你的用户问题计算向量相似度去库里扫前 5 相似的段落扔进了大模型。
*   **致命缺陷一：Lost in the Middle**：
如果找出来的相似片段极长（2w字）。根据所有学者的测试：不管什么神仙模型，只要你的致胜关键线索不幸掉落在这 2 万字的“正中间肚子部位”。大模型就会如同得了白内障一般大概率直接忽略！大模型有着极度偏向提取首尾两侧的畸形习惯（U型注意力坍塌曲线）。
*   **致命缺陷二：缺乏大局观 (Global View)**
如果问题是：“帮我归纳这几十份卷帙浩繁的公司年报主旨？” ，用普通 RAG，因为问题没有带明显实体靶子，相似度提取算法抓耳挠腮找不出相似关键词，提取会变成彻底瘫痪！

*   **GraphRAG 上下位碾压提取** (`graph_rag_retrieval.py`):
微软 GraphRAG 框架通过直接废除初期的 Vector 比较。改用雇佣无数只底层大模型做劳工蚂蚁，把十几万年报里的机构、金额和合同抠出来。结结实实扎了一张点和线连接的网络大图。
使用古典强悍的 **社区发现图算法（Community Traversing）**！把紧密的大图切割出比如：中国分公司簇、亚太对冲簇 等宏观聚点结构！
最后对这些粗颗粒度的群体各自总结发送给你的主大模型作答全局！这就是真正跳脱了上下文短视眼局限的神级图网降维打击。
