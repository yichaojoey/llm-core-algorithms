# Agentic Code Harness (SWE / 代码智能体底座)

> **面试一句话绝杀**：不要把 Agent 想成什么超级魔法。它就是一个**被洗脑且被严格规训只会吐出 JSON / XML 格式的带壳大模型**。我们后台系统会拦截它吐出的这个 JSON，跑到极其封闭安全的 Docker 沙盒环境里去帮它执行 Linux 终端命令，然后我们把终端上红油油的报错截取下来，反手扔给这个模型看。模型看到报错不断地自我纠正，最后成功了才给终端用户看。这就是所有的 Claude Code / SWE-Agent（也包括我能够自主帮你写库）的本质。

### 一. 系统拦截：强制死板调度 (Tool Orchestration)

> 参见源码模型：`tool_orchestration_loop.py`

让语言模型具备物理干涉能力的第一步？封死它说话的功能！
在 System Prompt 里塞下一堆极度威胁性质的命令，让它无论发生什么，想干活儿就只准输出 `{"name": "run_bash", args: {"command": "ls -l"}}` 类似的文本块。
在最最核心的调度层（Orchestrator），有一个死循环 `While True:`。模型只要在这个块里输出了这玩意，调度器直接死死掐住拦截切断输出！然后调度器后台拿着这些参数在真实操作系统里跑完这个 Python 原生函数，把得到的文本（比如文件夹列表）硬塞回给大模型作为 `[Observation]` 观察输入！这就相当于赋予了大模型“双手”。

### 二. 视野管控：视口分页拉取与精准 AST 替代

> 参见源码模型：`viewport_file_editor.py`

为什么大模型不能像我们在 IDE 里一样直接修改一万行的代码库改一两个 Bug？
因为即使可以输入 128k 代币，但是让它“输出（Output）”极长代币是天文级别的天价成本而且非常非常极度容易遗漏任何一个标点符号报废全局文件。
*   **Viewport (分页视口看码)**：绝不通过读取整个文件看，强制让模型调用带 `[1:800]` 这种行数起止坐标的工具指令来分页一块一块视察文件。
*   **AST Diff Replace (差异替换术)**：智能体最恐怖的能力！要求其提交修改请求时，极其苛刻地传出在 852行 和 853 行的特定原样字符串（Target），以及新的替换字符串（Replacement）。我们的后台系统在帮它去文件动刀子前，先执行全字符串完全比对校验。如果多一个空格直接驳回重试！这种变态要求极大避免了大模型直接冲垮线上核心祖传屎山代码。

### 三. 无限循环永动机：Bash 反馈自愈大阵

> 参见源码模型：`bash_feedback_loop.py`

智能体被尊为真神的核心点就是：人类睡觉的时候，它是如何在黑盒深渊里自己纠正错误的？
当智能体信心满满地改掉了你的核心库，然后它用刚才提到的工具调用了 `pytest tests/` 发现全部飙红炸开！
我们底层的 `subprocess` 极度精准地截获了这串包含行号的 Python 报错 Traceback...
我们不把这报给人类，而是再次循环静默把这段猩红色的 Traceback 串喂回给刚才乱写代码狂妄的 Agent！由于它极其强大的逻辑推理修复能力，立马认怂给出下一套打补丁修改块（参考第二点），修改然后再试！由于极其长远的思考循环不需要人类按哪怕一次确定的回车键，这种闭环构建了真正意义上的 **Auto-Healing (自愈式编程)** 帝国！
