# Decoder Block 动物园：大厂核心基座对比录

> 当一切 Attention、RoPE 算法写完后，要把它们像叠汉堡一样叠在一起。每个大厂都有自己不同的“夹心饼干”堆叠哲学。

这个文件夹包含了当今世界最顶级的三大流派：`LLaMA 3.1` (极简派), `Qwen 2.5` (特调派), `DeepSeek V3` (机械神教)。

---

## 🆚 面试核心决胜局：三家为什么这么造模型？

### 1. `LLaMA-3.1`: 纯净极简，一剑破万法
> **代码指路**：`llama3_1_block.py`
*   **特性**：通篇找不到一个 `bias=True`。采用极其经典的 MHA / GQA 搭配 SwiGLU。
*   **面试暴击**：为什么要做到 Zero-Bias (零偏置)？
    *   **Scale Invariance (尺度等变性)**：模型如果在某些极端高维数据里缩放数据特征，偏置会打破线性映射的纯粹性，极其容易造成大参数缩放溢出崩溃。
    *   极大方便了后期的量化（Quantization）：所有的权重全是硬核相乘，没有后续加法污染。

### 2. `Qwen-2.5`: 特修斯之船，处处小心机
> **代码指路**：`qwen2_5_block.py`
*   **特性**：非常逆天傲骨地把 Attention 的 Query、Key、Value 的 `bias` 全开了。保留了庞大多样的配置选项。
*   **面试暴击**：LLaMA 去掉了 Bias，为什么 Qwen 非要自己去捡回来？
    *   阿里团队的研究证明：如果拿掉了 QKV 的偏置，模型对于“数字、格式符、特定标点缩进”的关注度会陷入崩溃失明。尤其在写 Code 和算数时候。Bias 提供了一个极其微小但在所有输入上永恒存在的常数锚点，大大强化了特殊排版指令跟随！

### 3. `DeepSeek-V3 / R1`: 压榨物理法则的机械赛车
> **代码指路**：`deepseek_v3_block.py`
*   **特性**：完全是个异类。抛弃常规的 Attention，转而组装自己发明的 **MLA (潜变量解耦注意力)**。抛弃常规的 SwiGLU 前馈网络，转而换成极度零碎的 **256 路线路的 MoE 路由架构** 搭配 1 个全职辅助。
*   **面试暴击**：这是为了解决什么问题？
    *   **彻底消灭显存墙与算力墙**：上半身 MLA 将 128k 恐怖长度占用的天文级缓存（显存吃紧）压缩 90%；下半身 MoE 将需要全长计算的天文级稠密矩阵计算量（FLOPs 瓶颈）压缩了极其可观的部分。双管齐下创造了中国模型只用极小成本碾压 GPT-4 的不烂神话。
