# 预训练架构心脏：MoE 混合专家模型面试指南

> **面试一句话绝杀**：大模型参数如果做到 1000B (千亿/万亿) 以上还能推理跑通吗？不可能的。就算你有几百张 A100 全卡护城，光把这么巨大的矩阵纯计算挨个乘一遍走个前向流程时间也爆崩了！
> MoE 提出：我不管你参数存货有多大，但在面对每次每个具体单词或句子进行推理使用时，**我只去当庭唤醒（激活计算）其中专门负责这块领域的那一小撮最牛逼的核心精兵（专家）！**
> 所谓的：**“常驻显卡的静态权重极为庞大，但在每一次流动计算时激活的动态权重极小！”**
> 这就是算力救星的最高表现形式 （Sparse 稀疏化）。

## 致命拷问：专家坍塌 (Expert Collapse)

面试必问：**“如果不加入 Load Balancing Loss 惩罚，你的 MoE 模型用纯梯度下降跑到最后，会出现什么严重的现象？”**
答：会退化为**单一专家包揽全场（专家坍塌）**。

1. **马太效应**：刚初始化的时候大家都一样随机笨。由于某种偶然，专家 2 稍微比别的专家在第一个 Batch 计算梯度得分好了一点点。
2. **万劫不复**：Router 门控网络十分功利，它立刻在下一次打分支配流向时给它偏袒打出了高分权重，导致进入专家 2 的流量更大了！
3. **彻底旱死**：专家 2 接了源源不断的数据流，疯狂吸收大量知识做大量反向传播。越来越聪明。而专家 3 专家 4 由于早期没被发到几句话，直接饿死，得不到任何梯度修正，沦为死权重。
4. **终局**：这等价于训练了一个单体模型带了一堆死寂僵尸参数。浪费了全部资源。

## 面试必考公式解读

为打破垄断，就必须给这帮资本家加收“反垄断重头交易税”！
我们要手工算出一个指标来逼迫网络去均摊流量。它由两部分乘积做底计算：

*   $f_i$ : 这一轮 Batch 切下里，到底有多大比例的文字词汇硬性被塞进了你 专家 i 的屋子里？（代表绝对物理分单比！）
*   $P_i$ : 门口门卫系统 Router 面对这批所有的总词汇，宏观上给你打了多高的分？（代表被偏爱的期许值！）

两者结合后相乘之和 ： $Loss_{aux} = N \times \sum (f_i \times P_i)$
*   **如何救场**：当这群大爷接活（分单比）非常均衡时，或者门卫给的偏好分数极度中庸均衡时，这个公式在数学期望里有着雷打波澜不动的唯一最小值点（即常数 1.0）！一旦有极其偏科极其严重的挤兑现象发生，这个值会突破 1.0 开始爆雷暴涨。这时候反向传播带来的极重罚款会让 Router 悬崖勒马，强行把接下来的活分散分出！

执行 `python demo.py` 以及 `pytest test_moe.py` 直接观测那道 1.0 完美平衡线的压倒性控场展示。
