# 分词器：BPE vs WordPiece 面试圣经指南

> **一句话解释**：大模型就像一台外星机器，它看不懂人类优美的连贯文字 `beautiful`，你首先要做的就是教它如何把这个词查成密码段，通常我们不能把它作为一个完整的字，也不能敲碎成英文字母。必须建立一个基于概率粘合的智能词典（Vocab）。

---

## 核心算法对比剖析

为什么 GPT 选择了 **BPE (Byte-Pair Encoding)** 而 BERT 选择了 **WordPiece**？
两者同样都是将零碎字符强行捏合成“词根”的算法。但是在合体规则的价值观上根本不同。我们通过这两大体系来看待：

### 1. BPE 的绝对频次霸权
**算法逻辑**：不管三七二十一，我在海量文档里查字典频率。看到 `('h', 'e')` 紧紧挨在一起的次数最高（绝对数量 Number Count 大！），不管你们还有什么牵扯，第一步规则我就是要把你们死锁定绑成大一个号的符号 `he` 加入我的词海中。
**缺陷**：非常喜欢招纳一些“恶心但高频”的无意义碎渣组合因为其庞大的基数拉高了总占比。

### 2. WordPiece 的高傲互信息法则 (Probability Math)
**算法逻辑**：它算的是 **Mutual Information 互信息分数**： `Score = count(AB) / (count(A) * count(B))`
**意义**：如果你是 `('i', 's')`，虽然你们相聚的时间很长（分子大）。但是你们单独出去浪的次数太牛逼了（分母巨大）！一算分数极其低下不堪重用！
如果你是 `('x', 'y')`，由于哪怕全宇宙仅相聚了 10次。但由于我俩单独从未离开过各自。分母极小！分数直接登顶为 1.0 的满分结合！
**优势**：在语料库极端不平衡的情况下极其能保全专业、少见长词汇词根。

## 代码落实中的致命考点区别
如果你在考场要写出这两段代码区别，最致命的 `For` 循环深处只有那么几行改动：

**BPE 合并打分代码**：
```python
best_pair = max(pairs, key=pairs.get)  # 毫无感情拿 value 最高的，结束战斗。
```

**WordPiece 合并打分代码**：
```python
for pair, count_ab in pairs.items():
    P_A = single_freqs[pair[0]]  # 计算孤独概率
    P_B = single_freqs[pair[1]]
    score = count_ab / (P_A * P_B)  # 加上宏观降噪系数惩罚
    if score > max_score:
         best_record = pair
```

只需要记住这一手理论除法，就能在大模型面试上谈笑风生。
