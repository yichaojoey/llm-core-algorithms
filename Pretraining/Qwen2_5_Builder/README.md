## LLaMA 门徒测谎仪

现在的大模型算法岗位面试官深知：天下 99% 的开源初学者是从研读 LLaMA 论文开始起步入门大模型（LLM）的。这就导致这帮人的架构观产生“唯一的绝对真理”。

一个极高频的刺刀问题是：“在 Attention 的 `Linear` (线性转换全连接) 层中，我们是不是一概不建议保留 `bias (偏置)`？”

**初学者会这样死得透快**：对！LLaMA 证明了抛弃 Bias 可以让算式的权重矩阵变得极度纯洁清爽，更容易统一也更好做推演的量化压缩运算。所以现代大模型根本用不到 Bias。
**Qwen 王者破局法**：错！虽然抛弃是大势所趋，**但在极其强悍能够做到全线超越和霸榜开源界的 Qwen (通义千问 2 / 2.5 家族) 中，它在极其敏感的核心中枢： 生成 `Query`, `Key`, 及 `Value` 的三大投影矩阵中，硬气地全部保留了 `bias=True`！** 阿里的论文团队论证了在这种极致复杂的长序列相对距离（他们最高冲到了极其恐怖的 1M 上下文极长推理）里，加一点点这种平移参数（Bias）对于提升模型对外推位置的容忍度和数学外延表现有着巨大的正向保底作用。

*(注: 该考点我们已经写进了 `qwen_block.py` 的源码预警注释中)*。

## 生死的玄关：Pre-Norm 串流之道 (Residual Architecture)

第二个直接让你白板代码被毙掉的点就是残差（Skip-Connection 翻墙相加通道）与清理器 `RMSNorm` 的摆放位置。

如果让你拼接，你这辈子都绝对不能写出如下顺序的公式：
*   ❌ 淘汰毒药法 `Post-Norm`: $x_{new} = Norm( Attention(x) + x )$ 
**(先让原本肮脏未洗的特征进入庞大危险的打分矩阵翻滚计算，然后再出来用 Norm 归一化善后)**。这是老古董原版 Transformer 或者更早时代的陋习，它会造成深得无穷多的 80 层的超大模型里梯度疯狂紊乱极其难以寻峰收敛。

*   ✅ 当代标准天花板 `Pre-Norm`: $x_{new} = x + Attention(Norm(x))$
**(你自身本源的水流 $x$ 直接奔向彼岸，但在分支流向计算黑体前，强制先用清洁剂 $RMSNorm$ 把你要被黑体摄入的水狠狠梳理干净镇压一次，算出来的异次元污染果实安全后，再倒向加入进最初的那股本源水流！)**。这极大程度隔离和锁死了每次爆炸的范围，让几百上千亿的庞大训练规模稳定在红线以内！
