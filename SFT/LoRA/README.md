# LoRA (Low-Rank Adaptation) 面试指南

> 📖 **一句话解释**：绝不能全量微调大模型那数以千亿级别的硕大原生权重矩阵 $W$，而是在它的“旁边”单独新修一条小路强行“外挂”串联成两个极小尺寸的低秩矩阵 $A$ 和 $B$ 当代理。训练时只跟他们俩要更新梯度，推理服务上线时算完后直接并入原厂矩阵中彻底消失，极其经济有效甚至被奉为神迹！

## 背景

全参监督微调大模型（Full SFT / FFT）对不管是 7B 还是 72B 参数级的模型而言，除了加载权重自身占用的海量显存外，伴随其一同唤醒启动的由于追踪梯度流而要求维护的更巨型的额外信息池数组（Adam Optimizer 的各个梯度累积动量状态）会狂暴地膨胀到整台集群都无法招架容纳（常能暴增至本体体积容量的3到4倍大小！）。这显然非多数公司业务能承担起的，且无法针对千千万万个 C 端用户作精妙个人向定制！

这个时候就有一些具有数学直觉的大佬提出假说依据：**大模型的权重矩阵在迁移或适应某个非常偏科的专属下游新领域新任务上所做表现时，决定其转向起决定要素的本质“本案内蕴秩 (Intrinsic Rank)”其实极其低下（可以认为是空间信息稀疏冗余）——你根本不必兴师动众每一列全都变盘洗牌更改，只需要找到降级后的那些主要发力点（主成分空间）牵动它，整个表现能力就会完美达成所念结果。** 这样也就促生了这个 PEFT（参数高效指令微调界）一哥的诞生。

## Step 1：理论知识

假设原始大模型的某个 `linear` 线性层的巨大承重底盘墙尺寸为 $d_{in} \times d_{out}$。其巨型参数量设为 $W$。
LoRA 干脆通过禁用掉网络传导开关把它冻死在那里（$W_{freeze}$），在它平行架构着去“另修栈道”构建了由两个苗条的矩形状数组连通相乘构成的并向通路小路：
- $A$ 矩阵负责把进入其中的大空间通道压缩抽象重分布：身材尺寸变为 `in_dim x r`
- $B$ 矩阵负责把接受进来的小规模核心信号升维还原放回去：身材尺寸变为 `r x out_dim`

> $r$ (Rank 秩的大小) 往往微若游丝远远低于主干维度底盘（比如 $r=8$，原底盘 $dim=4096$）。

这也就代表，在实际训练打分算惩罚纠错微调的时候我们全盘抛开主架构不去管了，动摇调转更新的仅仅就是是这两个苗条精巧的轻装上阵矩阵 $A$ 和 $B$！此时数学上有加和合并关系：
$$ \Delta W = B \times A $$
$$ h = W_{old} x + \frac{\alpha}{r} (B \times A) \times x $$

## Step 2：代码实现

```python
class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, r=8, lora_alpha=16):
        super().__init__()
        self.r = r
        # 高频考点：缩放银弹参数，必须带着
        self.scaling = lora_alpha / r
        
        # 被剥夺求导权的硕大冻结主脉体 W
        self.weight = nn.Parameter(torch.empty((out_features, in_features)))
        self.bias = nn.Parameter(torch.zeros(out_features))
        
        # 负责全场游走、吸纳学习经验而求知若渴的活跃生动体 A 和 B
        self.lora_A = nn.Parameter(torch.empty((r, in_features)))
        self.lora_B = nn.Parameter(torch.empty((out_features, r)))
        
        # A 使用标准打散抛洒的 Kaiming 等价高斯随机性去初始化探索源头
        nn.init.kaiming_uniform_(self.lora_A, a=math.sqrt(5))
        # 🌟必背点：B 必须无情地硬塞压死为一堆全都是 0 的实心铁疙瘩！
        nn.init.zeros_(self.lora_B)

    def forward(self, x):
        # 保持骨干的自转计算流向不受阻塞
        original_output = F.linear(x, self.weight, self.bias)
        
        # 用巧妙结合律括号把 `x@A^T` 先做第一次减压降位算一次（极其省时间）再去拿小包结果去撞第二个 `... @ B^T` 回升回去
        lora_A_out = F.linear(x, self.lora_A)
        lora_B_out = F.linear(lora_A_out, self.lora_B)
        
        # 这里最后挂一个融合比例就完成了平行通道的输出合并，进入上层或下层的继续发力区块！
        return original_output + lora_B_out * self.scaling
```

## 面试必考追问

**Q: 为什么 $A$ 用随机高斯分布初始化了以后，反而后面那个起终点作用的 $B$ 必须设定为纯纯的 0 矩阵？这么做有什么非凡意义？**
> 这是 LoRA 在初始上车发车前绝对不会带跑带偏把模型原本优良语感炸毁破坏掉的“起航保护系统保证”：正是只有强制压死让 $B=0$ 纯零零落落，才让一开始处于零世代迭代下的初始加注偏差值恒挂为纯静态大零蛋：$\Delta W = 0 \times A = 0$。即保证了**你在介入并粗暴嫁接它上大模型并且拉开闸刀轰出第一步前推敲定的原始预估输出结果判决与不挂上它的原始巨头一模一样毫无二心！** 极其润滑无损地带过冷启动摩擦期缓冲保护大基座本体原厂预训练那份昂重浩瀚的绝佳知识常识底盘不产生剧烈颤抖式裂缝破坏！而不用害怕被突然挂空档初始拉出的突兀随机偏倚打崩全盘的知识图流结构（因为随着慢慢进入正式推导训练状态反馈发作，$B$ 上面的 0 就自然渐渐吸收了真实经验惩罚常量拥有了正式参数变动拉扯起了真身了）。

**Q: 代码段最后那个乘以前插的 `Scaling` 缩放因子常量（也就是 $\frac{\alpha}{r}$）是哪门子名堂玄学？我如果干脆直白点去在训练里不管不顾硬炼 $B \times A$ 合并体怎么讲？**
> 对于各不相同任务，甚至不同尺寸容量的网络它对各种不同的秩 $R$ 的敏感响应也是截然跳脱不一的。有由于多除了个这个天然平滑防浪涌的缓冲控制器存在去坐阵定标，你在面对某个复杂恶心或者简单的领域实验任务上甚至能够放心大胆地大手一挥随随便便将本来设置抠唆捉襟见肘的微薄低秩参数设置 $R=8$ 拔高暴增跳水调整放大为 $R=64$ 或以上——而且全程压根完全心不慌气不乱，完全不必再去手重调甚至提心吊胆的去微缩重新拉紧改变配置原来那个极度玻璃心高敏易碎的原有的全局的学习率超参数（Learning Rate）跨度调整了（否则一旦参数增多数倍很容易学习率过火跑飞扯步子撕挂整个版图）。$\alpha$ 一般是你在此先抛下拍脑门定性的经验常数，多数早期业内标准论文里会常设给出诸如 `16`、`32` 供大家拿直接抄作业起跑用的建议比例常量尺标！

**Q: 把它训练完成结束要封包送云端去对外客户跑服务（Inference 模型部署上线时刻）的时候，难道每一次我前向运算一个结果给客户时我底层计算里依然每次都还蠢蠢的多跑出来这么两条平行计算慢吞吞的多走绕圈打转算一车浪费算力的乘法岔路吗？**
> **完全不用！！！** 这也是我们经常高悬称赞表扬说这个玩意对比传统附加插件参数模块（Adapter Plugin类等家族算法）来说天然自带极其强大的免部署开销特征甚至真正被封神推广到满天下人人乐道所广泛吹捧喜爱接受到神坛层面的底层绝美核心原因。
> 因为：这里存在着在纯粹底层数学规则公式层面一次最高级优雅魔法戏法化用推导降维合并动作：“重参数化同构 (Reparameterization Merge / Weight Fusion Operation)”。在经过九九八十一难各种反向千丝万缕试炼获取出我们历尽千辛求取的绝美定死常数微调大成了后的这个结果并决定停止关机封装备产的时候，由于公式 $(W \times x) + (B \cdot A \times x) \cdot \frac{\alpha}{r}$  完全等效于并在数学上根据基础的高中分配连接加法律可以直接简单提出等同合并写为： $$(W + B \cdot A \cdot \frac{\alpha}{r}) \times x$$ 所以我们工程师人员压根只需要在上船发车之前，拉个大半夜系统极为空闲歇脚的低配 CPU 主机进程对静止的文件里跑上一小段仅仅运行那么一次常数合并归宗覆盖加法将 $W_{fuse} = W_{old} + B \times A \times (\frac{\alpha}{r})$ 加在文件老阵盘大骨架本体上更新一遍它自己的原参数覆写保存。**接着后续！接着客户请求推理时候就可以干干脆脆一把火全扬了烧扔了这俩碍事在后面慢腾腾碍眼的所谓临时施工辅修脚手架A与B副小模块通道甚至可以直接将它们清出内存全删得毛都不剩骨灰都不留痕迹跑出一个速度体积甚至一模一样没有一毫秒损耗完全不留岁月微调额外痕迹完全隐形的“极净快速无双纯血真微调”大模型本体去跑了！！！**
