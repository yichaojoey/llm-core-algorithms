# AdaLoRA (Adaptive Low-Rank Adaptation) 面试指南

> 📖 **一句话解释**：原装 LoRA 有个致命缺点：你得事先像瞎子一样猜测给每一层分配到底多粗的管道（秩 $R=8$ 还是 $16$）。但对于极其聪明的大模型来说，可能最后一层特别需要知识，最前面底层只需要皮毛。AdaLoRA 顺水推舟，在中间加了个“门卫打分器 $E$”，哪个通道不干活打分低直接当场抹杀节约预算，把容量留给最吃力的阶层使用！动态收紧。

## 背景

如果你微调过大模型你会发现：底层（前几层）在提取非常粗糙的 Token 和位置信息，而网络结构越往深走（接近最终解码层的深层 Transformer Block），它处理的高阶语义逻辑越复杂变态。
这就暗示我们：如果全线铺设普通 LoRA 时，给每一层都公平配额设置 `r=8` 是巨幅的人力智商浪费！前几层只用到 `r=2`，后面急得跳脚需要 `r=32`！但如果你手工一层层去定又根本没那神仙推算功夫。
于是 **AdaLoRA** 利用了近似数值数学里的神器 **SVD (奇异值分解 Singular Value Decomposition)** 概念，做出了极其出色的成绩碾压！

---

## Step 1：理论知识

经典 LoRA 将权重修正表示为两个小矩阵乘： 
$$ \Delta W = B \times A $$
但这两个黑盒子相乘后你完全分不出里面哪些成分是主力军，哪些是凑数滥竽充数的废品。
AdaLoRA 直接将其打撒变成三体结构，人为显式插入一个**打分过滤漏斗** $\Lambda$ (代码中叫 $E$)：
$$ \Delta W = P \times \Lambda \times Q $$
- $Q$：去提取抽象信息。
- $P$：重新排布送入大网流向。
- $\Lambda$：极其重要的中心角色！！它是一个非常小的数组（代表对角），数组里的每一个元素都记录着这个特定维度的特征流是否“具有大局显眼主导性”。如果它只有 0.0001 打分哪怕你 P 和 Q 干得热火朝天，这个结果一乘这数值也要变为粉齑。

## Step 2：代码实现

```python
class AdaLoRALinear(nn.Module):
    def __init__(self, in_features, out_features, r=8):
        # 1. 提取器 Q
        self.lora_Q = nn.Parameter(torch.empty((r, in_features))) 
        # 2. 🌟 判别核心漏斗 E
        self.lora_E = nn.Parameter(torch.empty(r)) 
        # 3. 释放器 P
        self.lora_P = nn.Parameter(torch.empty((out_features, r))) 

    def reset_parameters(self):
        # 防死锁核心考点（见下面问题 1 解答）
        nn.init.normal_(self.lora_P) 
        nn.init.normal_(self.lora_Q) 
        nn.init.zeros_(self.lora_E)  # 将把门不破坏大网络的纯真寄托交给 E。

    def forward(self, x):
        original_output = F.linear(x, self.weight, self.bias)
        # 用结合律: x 撞向 Q -> 用 E 一把拦下审查缩放(逐对角元素省去巨大乘法操作直接用内积挂靠) -> 最后送去 P
        lora_Q_out = F.linear(x, self.lora_Q)
        lora_E_out = lora_Q_out * self.lora_E.view(1, 1, -1) #(处理好广播)
        lora_P_out = F.linear(lora_E_out, self.lora_P)
        
        return original_output + lora_P_out * self.scaling
```

## 面试必考追问

**Q: 初始化时，普通 LoRA 让 A 随机，B 给全 0 来避免初始改变模型权重。那是不是 AdaLoRA 也可以随便把 P 或者 Q 其中一个给全 0 呢？为什么代码只让中间的 E 给 0？**
> **这是一个极高明最绝杀的顶级面试代码坑点，很多人手写绝对卡死在这里被面试官判死刑！**
> 如果在 AdaLoRA 这个三级流控矩阵中：你让放出端 $P$ 给定纯零初始值，并且为了让守门人也低调你由于潜意识顺带给了守门人 $E$ 纯零初始。当反向传播启动，框架追溯求 $E$ 的梯度时根据反向链式偏导乘法则：$\frac{\partial L}{\partial E} \propto  P^T \cdot ...$。
> **你会发现什么？如果 $P=0$！传向 $E$ 的救命恩人学习指导梯度也会常年被乘以这个硬性的 0 从而永世不得超生无法拿到丁点反馈指导！E彻底死锁冻僵学不到任何常识纠偏，永世为 0 然后把全网封死成植物人！**
> **正解**：为了保持发车时那宝贵初衷“模型不被突兀歪门拉低”且“网络得成功吸纳反馈破冰”，唯一的方法就是让 $P$ 与 $Q$ 都存在各种充满生机的乱数初始杂音正常高斯分布，唯独只用 $E=0$ 的阀门卡着最终融合输出的大门不让突兀杂音过去毁坏大架构！第一步梯度袭来时 $P,Q$ 是活生生的数值从而将珍贵的误差精准算盘反馈到 $E$ 头上，$E$ 破冰复苏。系统正式活化！这就是巧妙之神。

**Q: 到底它是怎么“自适应调节丢弃”的，具体表现是什么？**
> 我们有了一个打分手册 $E$！模型边训练边学边评估该维度通道数值的高低强弱。我们只需在训练大轮中插入一个“体检清查”。如果代码查出 $E$ 中部分向量（Rank）居然连 `0.01` 都没有混天度日的废柴，我们在内存库里直接利用一刀切乘法 Mask 技术强制把其对应矩阵行全量斩下洗为 $0$。那在这段参数显存就被释放节约出了资源用来挪动到需要它的深度 Transformer 去增添扩充，完全达成了动态分压配比目的！
